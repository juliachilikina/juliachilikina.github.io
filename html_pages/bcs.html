Bootcamp Spot

redesign

Bootcampspot is an educational space for UC Berkeley Extension students and faculty. Students access this site to check-in to class, submit homework, check grades, view upcoming workshops and check their peogress. We are a team of three designers and being bootcampers ourselves we'd been using BCS for 6 months and know its weaknesses. We found it ironic that a UX course has a website that clearly demonstrates lack of design. So for our final project, we decided to redesign bootcampspot.

We wanted to create a platform that is focused on the user experience. With all of the useful info on BCS, we want it to be the tool that students use for more than just checking in and submitting homework.

Type

UX/UI, web design

Time

2 weeks

My role

UX/UI designer

Overview

The current layout of the site (on the right) is not optimized for the tasks that are performed most frequently. We want to address this problem by redesigning the structure and layout of bootcampspot. Creating better information hierarchy, user flow and visual design will icrease the student satisfaction with using the platform.



Goals

  • Find user pain points

  • Understand which features students access the most and what are their expectations from the student platform

  • Define information architecture

Current version of BCS website

Website is not optimized for students needs, users have difficulties performing basic tasks because of the platform's lack of user focused navigation and structure.

Redesign BCS by improving the information architecture, UI elements, and navigation that will give students a delightful experience during their Bootcamp, so they can focus on their main goal of finding a career.

Research

Research questions

  • What tasks do users complete in bootcampspot?

  • How satisfied are users with the current layout and functionality?

  • What frustrations and difficulties do users face using BCS?

  • How often do users access the site?

  • How often do users attend workshops and use career services?




Heuristic evaluation & UI annotations

Evaluating the current layout and UI design was helpful in understanding what areas of the platform need improvement, and where it can prevent users from completing tasks in an inefficient manner. Thea main problems that were found are:

  • Unclear information hyerarchy

  • Lack of visual accents, page is difficult to scan

  • Repetitive information and empty seccions

  • No differentiation between primary and secondary CTA

Design system

The design system consists of the fonts, colors, interface elements, and sample imagery that communicate the essence of our visual brand. We wanted to create a platform that invites user to stay and explore. We use a super family font Poppins for 2 main reasons: Itâ€™s a geometric sans-serif that has big round characters and good spacing. Its big lower letters relative to the uppercase keeps the font from being boring and what is more important gives high readability. And, we wanted to use a Google font because it works well on almost all platforms. UI elements and components include primary and secondary buttons, toggles, cards and a set of icons I created.
